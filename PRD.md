# ShopGenie（商店精灵）— PRD

> 版本：v1.2 | 日期：2026-06-15 | 状态：V1.5 可用，运营闭环首版迭代中

---

## 1. 项目概述

**产品名**：ShopGenie（商店精灵）

**一句话描述**：面向电商商家的 AI 垂类 Agent，帮商家搞定内容创作、店铺运营、数据分析等一系列电商工作。

**产品愿景**：不是一个单一功能的工具，而是一个全能的电商 AI 搭档。商家跟它对话，就像跟一个懂电商的同事聊天——需要什么直接说，它帮你搞定。

**当前产品阶段**：
- **V1 / V1.5 已完成首版**：多平台内容生成、商品事实、记忆、内容资产与平台强契约。
- **运营闭环已完成首版**：每日运营指挥台、评论反哺、A/B 实验、效果数据、CSV/平台 API 只读回流、营销日历与发布排期。
- **提前验证能力**：客服话术、公开竞品/趋势来源发现、批量生成。
- **暂缓能力**：商品图工作室已从前端下线，不在当前产品范围；后端实验代码暂保留。
- **下一阶段**：真实商家内测、内容质量抽检、账号鉴权与数据隔离。

## 2. 目标用户

| 用户画像 | 描述 |
|---------|------|
| **主要用户** | 中小电商商家、品牌方运营、带货达人 |
| **典型场景** | 每天要发内容（短视频/笔记），内容创作是刚需 |
| **痛点** | ①创意枯竭，不知道拍什么写什么 ②每个平台风格不同，手动适配累 ③招内容运营成本高 |

**暂不覆盖**：大品牌（有专门团队）、代运营公司

## 3. V1：电商内容生成

### 3.1 核心功能

V1 专注一件事：**帮商家产出能卖货的内容**。

```
输入：商品信息 + 目标平台 + 想拍/写的类型
  ↓
ShopGenie 处理
  ↓
输出：可直接使用的内容（脚本/笔记/文案）
```

### 3.2 支持的内容类型

| 平台 | 内容类型 | 产出物 |
|------|---------|--------|
| **抖音** | 短视频脚本 | 完整脚本（hook开头 + 产品展示 + 卖点讲解 + 引导下单） |
| **抖音** | 商品文案 | 抖音小店标题 + 商品详情 |
| **小红书** | 种草笔记 | 图文笔记（标题 + 正文 + 标签） |
| **小红书** | 测评文案 | 专业测评风格的文章 |
| **Amazon** | 商品 Listing | 英文标题 + 3-5 条 Bullet Points + 商品描述 |

当前已支持抖音短视频脚本、抖音小店商品文案、小红书笔记、Amazon Listing 和客服话术。小红书测评文案仍复用小红书笔记生成能力，尚未建立独立内容类型契约。

### 3.2.1 平台强契约与自动矫正 [MUST]

用户创建会话时选择的平台，是该会话最终成品的唯一平台来源。用户可以在对话中讨论其他平台，但 Agent 不得因为用户粘贴的 Prompt、历史消息或示例而静默产出其他平台格式。

当用户在当前会话中明确要求生成另一平台内容时，Agent 必须先询问用户是：

1. 将需求转换为当前平台内容；
2. 新建目标平台会话后生成。

成品展示前必须经过确定性结构校验，不依赖 LLM 自行判断：

| 平台 | 最低合格结构 |
|------|-------------|
| **抖音短视频脚本** | 至少 3 个分镜；每个分镜包含明确时间段、画面/镜头说明、口播；包含开头 Hook 和结尾转化动作 |
| **抖音小店商品文案** | 包含可直接上架的商品标题、完整商品详情、至少 2 个卖点/场景段落；不得输出分秒脚本 |
| **小红书** | 包含标题、自然分享型正文和标签；不得输出分秒脚本或 Amazon Listing 结构 |
| **Amazon** | Listing 成品使用英文；包含 Title、3-5 条 Bullet Points、Description；不得输出中文笔记或短视频分镜 |

结构不合格时，系统自动携带校验错误矫正一次。矫正后仍不合格，则返回明确失败提示或待补充草稿，不展示错误平台的成品。跨平台格式污染必须有回归测试覆盖。

### 3.3 对话交互

ShopGenie 以对话方式工作，不是填表单。

**示例 1：抖音短视频脚本**

```
商家：帮我写个面膜的抖音脚本

ShopGenie：好，几个问题：
1. 这款面膜主打什么？（补水/美白/修护？）
2. 你想拍什么类型的？（上脸实拍/成分讲解/对比测评？）
3. 时长大概多少秒？

商家：玻尿酸补水面膜，想拍上脸实拍，30秒左右

ShopGenie：收到。你店铺风格偏真实感还是偏精致感？

商家：真实感，不要太广告

ShopGenie：好，给你生成了：

【30秒短视频脚本】

🎬 开头hook（0-3秒）
"姐妹们，我终于找到干皮的救星了！"
（镜头：手拿面膜，表情惊喜）

📦 产品展示（3-8秒）
"就是这个XX玻尿酸面膜，你看这个精华液——"
（镜头：挤出精华液展示质地）

💧 上脸过程（8-20秒）
"直接敷上，冰冰凉凉的，敏感肌也没刺痛感..."
（镜头：敷面膜过程，展示服帖度）

✨ 效果展示（20-25秒）
"15分钟后，你看这个水润感——"
（镜头：摘面膜后皮肤特写）

🛒 引导下单（25-30秒）
"链接在下方，今天下单还有买二送一！"
（镜头：指向购物车图标）

---
💡 拍摄建议：
- 自然光，不要打太强的灯
- 语气真实自然，像跟闺蜜聊天
- 开头3秒是关键，表情要到位

要调整哪里直接说。
```

**示例 2：小红书种草笔记**

```
商家：帮我写个小红书笔记，也是这个面膜

ShopGenie：好，你想写哪种风格？

2. 成分分析型（"扒一扒这个面膜的成分"）
3. 合集推荐型（"我的换季护肤清单"）

商家：真实分享型

ShopGenie：好：

正文：

- 首图用敷面膜的对比图（敷前/敷后）
- 晚上8-10点发布，流量最好

要改哪里直接说。
```

### 3.4 记忆系统

| 记忆层 | 内容 | 生命周期 |
|--------|------|---------|
| **品牌档案** | 品牌名、主营品类、品牌调性、目标人群 | 永久 |
| **风格偏好** | 内容风格（真实/精致/搞笑）、常用平台、禁忌词 | 永久，可更新 |
| **会话上下文** | 当前正在处理的商品、对话内容 | 单次会话 |

记忆来源：
- 首次使用引导填写（可跳过）
- 对话中自动提取并记忆
- 用户主动修正
- 用户手动管理

### 3.5 后处理

内容生成后自动执行：

1. **违禁词检测** — 平台违禁词（抖音/小红书/Amazon 各有不同）、广告法违规词
2. **平台规则检查** — 小红书不能太硬广、抖音开头必须抓人、Amazon Listing 避免促销信息与关键词堆砌等
3. **标签/话题建议** — 根据内容自动推荐热门标签

### 3.6 质量铁律

- **前端必须好看** — 打开就有产品感，参考主流 SaaS Chat UI
- **输出必须能直接用** — 生成的脚本和笔记，商家复制就能用，不给废话
- **成本必须可控** — 一次完整生成（脚本+笔记）控制在几分钱以内

## 4. 运营增长闭环（首版已实现，持续迭代）

| 功能 | 说明 |
|------|------|
| **Listing优化** | 抖音小店/淘宝/拼多多/Amazon 的商品标题、详情页深度优化 |
| **标题SEO** | 关键词分析、标题评分、搜索排名建议 |
| **竞品分析** | Agent 自动搜索同类竞品，分析其文案策略、价格区间、卖点差异，给出差异化建议（不需要用户手动输入竞品信息） |
| **多平台适配** | 同一商品一键生成多平台适配的文案 |

V2 跟 V1 的区别：V1 是"帮你想内容"，V2 是"帮你优化店铺"。竞品分析由 Agent 主动搜索，用户只需要告诉 Agent 自己的商品信息，Agent 自己去网上找竞品来分析。

### 4.1 每日运营指挥台（V2 首版）

**目标**：用户打开 ShopGenie 后，不需要先思考要用哪个工具，直接看到今天最值得做的 3-5 个运营动作。

**首版输入**：
- 商品事实与评论洞察是否完整；
- 内容资产是否已有发布效果；
- 单次发布的曝光、点击和转化漏斗；
- A/B 实验是否仍缺真实投放数据。

**首版确定性诊断规则**：

| 信号 | 建议动作 |
|------|----------|
| 尚未建立商品 | 先建立商品事实卡 |
| 商品尚无评论洞察 | 导入真实评论，提炼用户卖点与顾虑 |
| 内容资产尚无效果数据 | 回填首次发布效果，启动数据闭环 |
| 曝光达到 100 且点击率低于 1% | 优先优化标题、封面或前三秒钩子 |
| 点击达到 20 且点击后转化率低于 2% | 检查卖点、详情页与转化表达 |
| 下单达到 10 且退款率高于 10% | 排查宣传承诺、商品事实和客服说明 |
| A/B 实验尚无曝光数据 | 投放变体并回填实验数据 |

**展示要求**：
- 建议按高、中、低优先级排序，最多展示 5 条；
- 每条必须包含大白话原因、关键指标和明确行动按钮；
- 点击行动按钮必须直达对应工作台模块；
- 无风险信号时展示明确的健康状态，不制造虚假焦虑。

**首版边界**：建议基于用户录入数据，不声称代表平台完整经营情况；不自动发布、不自动修改内容、不执行高后果操作。

### 4.2 运营能力演进路线

> 路线用于保存方向与依赖，不代表全部立即开发。状态分为：候选、已排期、开发中、已完成。

| 优先级 | 能力 | 用户问题与价值 | 成功指标 | 依赖 / 首版边界 | 状态 |
|--------|------|----------------|----------|-----------------|------|
| **P0** | CSV 数据导入 + 完整漏斗 | 手填数据成本高，当前只有粗粒度曝光转化，诊断容易失真 | 可批量导入；错误行明确；漏斗指标可用于建议 | 首版支持标准 CSV 与内容资产 ID，不接平台 API | **已完成** |
| **P0** | A/B 最小样本量与置信提示 | 当前少量曝光也可能过早判定赢家 | 不足样本不宣布赢家；展示可信度提示 | 后端确定性门槛与前端提示均已完成；不承诺统计显著性 | **已完成** |
| **P1** | 内容日历与活动素材拆解 | 商家不知道每天发什么，活动素材准备容易遗漏 | 每周计划完成率、活动素材覆盖率 | 营销节点、选题入口与发布排期首版已完成；素材覆盖率与计划完成率待补 | **已完成首版** |
| **P1** | 评论负面趋势预警 | 差评问题通常发现太晚，内容和客服无法及时修正 | 高频负面问题发现时间、重复咨询率 | 依赖持续导入评论；只做提示，不自动改商品承诺 | 候选 |
| **P2** | 竞品持续监控 | 单次竞品搜索无法发现价格、卖点和内容趋势变化 | 有效变化提醒数、差异化内容采用率 | 仅使用公开来源并保留来源；不依赖脆弱自动发布 | 候选 |
| **P2** | 跨商品、跨平台诊断看板 | 商家无法快速判断资源应投向哪个商品和平台 | 找到高潜商品所需时间、建议采纳率 | 依赖稳定的数据导入与统一指标口径 | 候选 |
| **P3** | 平台 API 数据连接器 | 手工导入仍有摩擦，数据不够实时 | 自动同步成功率、数据延迟 | 配置驱动的只读连接器首版已完成；实际平台授权与接口适配由部署配置提供；写操作不在范围内 | **已完成（只读首版）** |
| **P3** | 自动发布与运营执行 | 商家希望从建议直接走到执行 | 节省操作时长、发布成功率 | 高后果操作，默认必须人工确认 | 候选 |

### 4.3 CSV 数据导入 + 完整运营漏斗（首版）

**标准 CSV 字段**：

| 字段 | 必填 | 说明 |
|------|------|------|
| `asset_id` | 是 | ShopGenie 内容资产 ID，必须真实存在 |
| `impressions`、`clicks`、`add_to_carts`、`orders`、`refunds` | 否 | 非负整数，空值按 0 |
| `revenue`、`ad_spend` | 否 | 非负金额，空值按 0 |
| `notes` | 否 | 运营备注 |

**导入规则**：
- 必须先预览校验，全部行通过后才允许导入；
- 任一行错误则整批不写入，并返回明确行号与原因；
- 单次最多 500 行，拒绝超大或空 CSV；
- 平台从内容资产契约读取，不信任 CSV 自报平台；
- 首版不去重、不覆盖旧记录，每次成功导入都新增一批效果记录。

**完整漏斗汇总**：曝光、点击、加购、下单、退款、销售额、广告花费、CTR、点击后成交率、退款率、ROAS。

### 4.4 前端体验优化方向

1. **收敛玻璃材质**：玻璃保留在侧栏、顶栏、悬浮输入区和弹层；内容卡、商品详情、效果数据使用稳定实色表面，拉开信息层级。

3. **工作台信息架构（已完成首版）**：商品、内容、实验、知识、任务、效果、营销日历均为一级路由，支持查询参数深链和移动端访问。

5. **统一状态语言**：为当前商品、草稿、已发布、异常、待回填建立一致状态标签；运营建议按严重程度、趋势和时间表达。

### 4.5 商品上下文强契约

**问题**：如果查看商品、用于生成、评论洞察和会话历史使用不同商品，模型会把 A 商品历史与 B 商品事实混在一起，产生不可用内容。

**规则**：
- 应用不得默认选中商品库第一条；商品必须由用户显式选择。
- 查看商品详情不改变当前生成商品。
- 会话产生第一条真实消息后，商品上下文锁定；切换商品必须新建会话。
- 评论洞察保存所属 `product_id` 与商品名，只能注入同一商品。
- 无归属的旧洞察停止注入，用户重新分析后恢复。
- 后端收到不存在或已删除的商品 ID 时明确拒绝，不得静默降级为无商品生成。

### 4.6 运营指挥台 v2：规则引擎升级与建议闭环（已完成首版）

> 当前已完成按资产聚合、平台差异化 CTR 基准、14 天窗口与近 7 天趋势、按曝光影响排序、建议状态持久化与带上下文的一级路由直达。高退款“宣传 vs 真实体验”差异报告仍未实现。

#### 4.6.1 规则引擎修正 [MUST]

当前 `operations.py` 已完成以下四项修正：

| 缺陷 | 后果 | 修正 |
|------|------|------|
| 按 PerformanceRecord 逐条触发诊断 | CSV 导入不去重，同一资产多条记录产生重复建议，5 个坑位被同一问题刷满 | 先按 `asset_id` 聚合（或取最新一条），再诊断 |
| CTR/转化率阈值全平台一刀切 | 小红书、抖音、Amazon 基准差异大，误报/漏报 | 阈值改为平台维度字典（参照 `ab_testing.PLATFORM_HINT` 的做法） |
| 无时间维度 | 三周前的旧数据与昨天的新数据同等告警 | 只诊断最近 N 天数据；优先输出趋势信号（"CTR 从 2.1% 跌到 0.8%"比绝对值有用） |
| 同优先级内按 action.id 字典序排序 | 排序近似随机 | 按影响排序：曝光量大的资产优先（1 万曝光的低 CTR 比 200 曝光的值钱） |

#### 4.6.2 建议生命周期 [MUST]

建议状态已持久化，当前前端支持忽略；后端支持 `open / done / dismissed`，新数据进入后可重新触发。

- 每条 action 增加持久化状态：`open / done / dismissed`；
- 用户可一键标记"已处理"或"忽略"，已处理的建议次日不再重复出现（除非信号再次触发且数据有更新）；
- 状态数据天然产出 4.2 已定义的成功指标"建议采纳率"。

#### 4.6.3 建议直达动作（建议 → 执行闭环）

每条建议从"跳转某个工作台 tab"升级为"挂一个预填上下文的直达动作"：

| 信号 | 直达动作 |
|------|----------|
| 点击率低 | 「生成 3 个新标题变体」：预填该商品 + 该资产上下文，直接调 `generate_variants` 落成 A/B 实验 |
| 退款率高 | 拉取该商品评论洞察的 `pain_points`，对照内容承诺生成"宣传 vs 真实体验"差异报告 |
| 缺评论洞察 | 直接打开该商品的评论粘贴框（而不是商品列表） |
| 内容无效果数据 | 直接打开该资产的效果录入表单 |

完整循环：诊断 → 一键生成变体 → 投放 → 回填 → 下次诊断验证是否改善。

### 4.7 数据回流减摩擦（评审新增候选）

所有诊断的质量上限取决于回流数据量；手填和 CSV 对中小商家都偏重。在 P3 平台 API 之前的两个低成本中间态：

1. **截图识别录入（高杠杆）**：商家最习惯的动作是截平台后台的图。复用现有 `vision.py`：用户截图小红书创作者中心 / 抖店罗盘 → 视觉模型提取曝光/点击/转化 → 预填表单供确认后入库。比等平台 API 现实得多。
2. **回填提醒**：内容资产标记"已发布"后第 3 天，指挥台自动生成"回填《xxx》发布效果"的建议——把回流本身纳入建议体系。
3. **按资产聚合展示**：效果数据页从"record 流水账"改为按 `asset_id` 聚合的卡片（累计漏斗四格 + 最近一次 vs 上一次的趋势箭头），流水账收进详情。

另一项高性价比候选：**一页式周报**——每周自动汇总发布数量、各平台漏斗对比、A/B 赢家及其策略角度（`angle` 字段已存）、下周建议，用 LLM 把结构化数据写成大白话复盘。对雇不起运营的中小商家，这是"周会汇报"级的情感价值与留存抓手。

### 4.9 UI 设计系统收敛（2026-06-12 评审）

> 现状判断：首页"杂志感"成立（暖灰纸感 + 点阵 + 衬线标题 + 平台 mock 预览有辨识度），但全产品实为两套 UI——首页是设计过的，工作台仍是"后台脸"；设计投入与使用时长倒挂（运营用户 80% 时间在工作台）。

#### 4.9.1 首页一屏原则 [MUST]

首页当前约 1400px 高、桌面需滚动近两屏，一个组件背了 5 个职能（平台预览卡、批量 CTA、指挥台、更多场景、品牌档案 chip）。**滚动不是空间问题，是定位问题**：日用工具首页应按"驾驶舱"逻辑排版，一眼尽收。硬约束：桌面一屏（~900px）放下。

```
┌─────────────────────────────────────────────┐
│ 问候 + 今日状态一句话（≈60px）                  │
├──────────────────────────┬──────────────────┤
│ 今日运营指挥台            │  运营资产三卡       │
│ 建议任务卡 ×3-5（主体，    │  商品库/实验/效果    │
│ 占左侧约 2/3 宽）         │  （纵排小卡）       │
├──────────────────────────┴──────────────────┤
│ 创作快捷行（≈80px）：                          │
│ [小红书] [抖音] [Amazon] [客服] [批量]          │
└─────────────────────────────────────────────┘
```

取舍：
- 三张平台预览卡是最大空间消费者，降级为一行平台色 chips；mock 预览（教学素材）挪到 hover 浮层或选完平台后的空聊天页；
- 批量 CTA 从全宽橙块降级为快捷行按钮；
- "更多场景"并入快捷行；品牌档案 chip 归入侧栏 profile 卡。

#### 4.9.2 Design token 清理 [MUST，半天工作量，先做]

- **色彩**：实际在用的强调色 ≥6 个（`--accent #e8462a` 与误命名的 `--ios-blue #ff7a5c` 两个橙、小红书红、绿、紫、琥珀）。收敛为：一个行动色（珊瑚橙，合并两值）+ 语义色组（成功/警告/危险）+ 平台色只许出现在 dot/pill 等 identity 标记；
- **字阶**：现状 9/9.5/10/10.5/11/12.5/13.5px 全谱系，中文 11px 以下不可读。定死 5 级（如 11/12/13/15/20），11px 只给装饰性英文标签；
- **圆角**：从 4 到 24 全谱系收敛到 3 档；
- **数字**：KPI 用已引入的 Space Mono 等宽数字 + 趋势箭头，数据要对得齐。

#### 4.9.3 工作台提升到与首页同级的设计水准

- 表单分组：商品详情按"基本信息 / 事实与卖点 / 合规红线 / 评论洞察"分 section，不再一列裸表单到底；
- 空状态设计：KPI 无数据时不得显示 `0% CTR · 0 ROAS`（看起来像坏了），改为引导式空状态（"导入第一批数据"）；每个 tab 都要有空状态；
- 状态 chip 组件化（呼应 4.4.5）：草稿/已发布/待回填/异常，列表、指挥台、详情页复用；
- 玻璃收敛执行 4.4.1：玻璃只留 chrome（侧栏/顶栏/弹层），内容用实色表面，消除玻璃叠玻璃的对比度问题；
- **数据表面视觉语言**：真实效果数据（实色、等宽数字、趋势色）与 AI 生成内容（卡片语言）在视觉上区分——运营工具的信任感来自"数字看起来是认真的"；
- 指挥台任务卡语言：左侧优先级色条 + 加粗标题 + 等宽指标 + 明确 CTA 按钮；入场动画收敛到 ~300ms 内，高频日用不让动画挡路。

#### 4.9.4 建议落地顺序

① token 清理（纯 CSS 变量层，全局观感立刻齐整）→ ② 指挥台规则修正 + 建议生命周期（4.6.1/4.6.2，小改动立刻提升可信度）→ ③ 数据层 + 路由化（4.8）→ ④ 首页一屏改版（4.9.1）+ 建议直达动作（4.6.3）→ ⑤ 截图识别回填（4.7）→ ⑥ 节点日历、周报。

### 多平台扩展优先级

扩平台前置条件：现有抖音、小红书、Amazon 三平台通过平台强契约、结构校验和跨平台污染回归测试。新增平台必须有独立 Prompt、确定性校验器、合规规则和结果预览，不以“复用相近平台 Prompt”作为完成标准。

| 优先级 | 平台 | 核心产出 | 选择理由 |
|--------|------|----------|----------|
| **P1** | TikTok Shop | 英文短视频脚本、商品 Listing | 跨境社交电商核心渠道，可复用部分抖音能力但语言、表达和规则不同 |
| **P1** | 视频号 | 短视频脚本、直播预热文案 | 国内商家覆盖广，偏信任与私域转化，区别于抖音强 Hook 风格 |
| **P1** | 淘宝/天猫 | 商品标题、卖点、详情页文案 | 国内店铺运营的主流需求，与 V2 Listing 优化高度匹配 |
| **P2** | Shopify / DTC 独立站 | 商品页、广告文案、邮件 | 跨境品牌商家常用，适合从内容生成延伸到转化优化 |
| **P2** | 快手 | 短视频脚本、直播带货话术 | 内容结构接近抖音，但更强调人设、信任和直播转化 |
| **P2** | 拼多多 / 京东 | 标题、卖点、详情页文案 | 商家规模大，但平台调性和商品信息结构差异明显 |
| **P3** | Etsy、eBay、Walmart Marketplace | 英文 Listing | 覆盖细分跨境卖家，在核心平台稳定后接入 |
| **P3** | Instagram Reels、YouTube Shorts | 海外短视频脚本 | 用于海外内容分发，后续与 TikTok Shop 内容复用联动 |

### V1.5：内容工作台（已实现首版）

V1 不再只返回一次性文案，而是围绕商家的商品和内容形成持续优化闭环：

- **商品事实库**：每个商品独立保存卖点、确认事实和禁止声明；生成时自动注入。
- **显式自动记忆**：用户在对话中使用 `事实：`、`卖点：`、`禁止：` 标注的信息，会安全写回当前商品。
- **内容资产与版本**：每次成品自动存档；支持手动编辑、版本历史、局部 AI 改写。
- **质量闭环**：确定性检查标题、正文、占位项、合规风险、商品禁用声明和平台结构；低分内容可按建议自动修订。
- **知识来源**：商家可录入或从公开网页导入已验证资料；用户明确要求最新规则、趋势、竞品调研时，Agent 自动执行有界网页发现，按需求检索并返回来源链接。
- **Agent 任务**：记录每次生成的执行计划、完成/失败状态；也可针对已有内容执行优化任务，自动创建新版本并重新质检。
- **效果数据**：记录发布后的曝光和转化，汇总整体转化率，并将同商品同平台的历史表现反馈到后续生成。

当前边界：公开网页发现依赖搜索结果和来源页面，仍需商家判断来源是否可信；效果数据支持手动、CSV 与配置驱动的只读平台 API 连接器。实际平台授权、字段映射与接口可用性取决于部署方配置。

### 4.10 平台 API 只读连接器（首版）

- 后端环境变量预配置平台 API 地址、只读令牌和指标路径；前端不能传入任意 URL 或令牌。
- 首版支持小红书、抖音、Amazon 效果数据同步，不执行发布、改内容、改权限等写操作。
- 远端记录必须携带稳定 `record_id` 与真实 ShopGenie `asset_id`；重复同步幂等覆盖，不重复累计。
- 同步前校验资产存在、平台归属和漏斗关系；任一记录失败则整批拒绝。
- 单次最多同步 500 条，令牌不会通过状态 API 返回前端。

## 7. 技术架构

### 设计原则

- **简单优先**：不过度设计，能一行解决不写两行
- **LLM 能力兜底**：不搞复杂 Router/编排层，LLM 自己判断该干什么
- **后处理兜底**：违禁词、格式校验用确定性代码，不靠 LLM 自律
- **逐步演进**：V1 最简架构，功能加了再扩

### 架构图

```
┌──────────────────────────────────────────────────────┐
│                    前端 (Next.js)                     │
│                                                      │
│ Chat UI | 一级工作台 | 用户档案 | 结果展示/编辑          │
└───────────────────────┬──────────────────────────────┘
                        │
┌───────────────────────▼──────────────────────────────┐
│                   后端 (FastAPI)                      │
│                                                      │
│   ┌─────────────────────────────────────────────┐   │
│   │              /api/chat                       │   │
│   │                                             │   │
│   │  1. 读取用户记忆                              │   │
│   │  2. 加载平台规则 (`prompts.py`)               │   │
│   │  3. 组装 Prompt + 商品/知识/效果上下文          │   │
│   │  4. 调用 LLM                                 │   │
│   │  5. 后处理（违禁词、平台结构、自动矫正）        │   │
│   │  6. 保存合格内容资产并返回                     │   │
│   └─────────────────────────────────────────────┘   │
│                                                      │
│   ┌──────────┐ ┌──────────────┐ ┌──────────────┐   │
│   │  Memory   │ │  Workspace   │ │ Platform Rules│   │
│   │  记忆存储  │ │ 商品/内容/效果 │ │ prompts.py     │   │
│   └──────────┘ └──────────────┘ └──────────────┘   │
└───────────────────────┬──────────────────────────────┘
                        │
┌───────────────────────▼──────────────────────────────┐
│                    LLM API                            │
│                                                      │
│   DeepSeek (主力)  |  MIMO (备选)                     │
└──────────────────────────────────────────────────────┘
```

### 核心循环

```
用户输入
  ↓
组装 Prompt（系统指令 + 记忆 + 平台规则 + 对话历史 + 用户消息）
  ↓
调用 LLM 生成结构化草稿或成品
  ↓
后处理（违禁词、平台结构校验；失败最多自动矫正一次）
  ↓
合格内容入库并返回；不合格则明确报错或返回待补充草稿
```

保持简单：生成链路直接组装上下文调用模型，不引入复杂 Router；公开来源发现、质量检查、效果诊断和平台同步由明确模块与确定性规则负责。

### 技术选型

| 层 | 选型 | 理由 |
|----|------|------|
| 前端 | Next.js + React + TypeScript + SWR + CSS | 静态导出、一级路由和统一数据缓存 |
| 后端 | FastAPI (Python) | 简单直接，LLM 生态库多 |
| LLM | DeepSeek V4 Flash | 中文能力强、成本低、OpenAI 兼容格式 |
| 数据库 | SQLite (V1) | 零配置，一个人用够了 |
| 平台规则 | `prompts.py` 中的 `PLATFORM_PROMPTS` | 平台规则与生成 Prompt 保持单一来源 |
| 部署 | Next.js 静态导出 + nginx；FastAPI/Uvicorn | 适配当前单机部署，后续再补进程托管与 CI/CD |

### 关键模块

#### PromptEngine

整个 Agent 最核心的模块。

```
System Prompt = 基础人设（ShopGenie 的角色和能力）
              + 用户记忆（品牌、偏好、风格）
              + 平台规则（从 `PLATFORM_PROMPTS` 加载，抖音/小红书/Amazon 不同）
              + 商品事实、知识来源与历史效果
              + 输出格式要求
```

#### 确定性能力模块

平台结构校验、违禁词检查、商品归属校验、效果数据导入、A/B 判胜和运营建议均由后端确定性代码执行。LLM 负责内容生成与改写，不负责替代这些业务契约。

#### Memory

```python
user_profile = {
    "brand_name": "XX美妆",
    "category": "护肤品",
    "tone": "真实感",
    "target_audience": "25-35岁女性",
    "platforms": ["douyin", "xiaohongshu", "amazon"],
    "style_preferences": ["不要硬广", "像朋友聊天"],
}
```

#### PostProcess

LLM 返回结果后、返回给用户之前：
1. 违禁词检测（正则 + 词库）
2. 平台规则检查（小红书不能太硬广等）
3. 标签推荐（根据内容自动匹配热门话题）

## 10. 竞品简析

| 竞品 | 优势 | 劣势 | 我们的差异 |
|------|------|------|-----------|
| 剪映AI脚本 | 平台内嵌 | 只覆盖抖音，功能单一 | 多平台 + 记忆 + 对话式 |
| 小红书官方AI | 平台内嵌 | 只覆盖小红书 | 跨平台 + Agent 能力 |
| 通用AI写作（秘塔等） | 通用能力强 | 不懂电商规则，不检测违禁词 | 电商垂直 + 规则引擎 |
| ChatGPT/DeepSeek | 能力强 | 没有电商记忆、不懂平台规则 | 垂类 Agent + 记忆 + 规则 |

**差异化**：跨平台电商 Agent（记忆+对话+逐步扩展）+ 垂直深度（平台规则+违禁词+标签推荐）
