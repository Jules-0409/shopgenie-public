# 交互逻辑测试用例

> 改动：选平台不立即建会话，发第一条消息后才出现在侧栏。

## 测试步骤

### T1：选平台不建会话

1. 打开 https://www.liujufu.com/shopgenie/
2. 点击「小红书种草笔记」卡片
3. **预期**：切到聊天界面，顶部显示「小红书」pill + 「新对话」
4. **预期**：左侧侧栏**没有**新增任何会话（只有演示会话）
5. **预期**：输入框可见，预填了 starter text

### T2：发消息后出现在侧栏

1. 接 T1，输入框已有文字，直接按 Enter 发送
2. **预期**：消息发出，开始生成
3. **预期**：左侧侧栏**立刻出现**新会话
4. **预期**：新会话标题从 LLM 返回的 conversation_title 更新

### T3：中途返回再选平台

1. 已有至少一个真实会话
2. 点「新对话」→ 回到欢迎页
3. 选「抖音短视频脚本」
4. **预期**：切到聊天界面，侧栏不变
5. 不发送，直接按「新对话」→ 回到欢迎页
6. **预期**：侧栏依然只有之前的会话，没有新的

### T4：切换已有会话

1. 欢迎页选平台但不发消息
2. 点侧栏中已有的某个会话
3. **预期**：正常切换到该会话，pending 状态清空
4. 再点「新对话」→ 回欢迎页
5. 选同样或另一个平台但不发消息
6. **预期**：正常切到空白聊天界面

### T5：连续发多条

1. 选平台 → 发第一条消息 → 会话出现在侧栏
2. 在同一个会话继续发第二条
3. **预期**：侧栏不变（不会重复创建）
4. 第三条、第四条同理

### T6：商品图工作室入口

1. 在欢迎页选择「商品图工作室」
2. **预期**：进入独立 Studio 页面，前端成功加载场景模板，不出现 API 404
3. 不生成内容，返回欢迎页
4. **预期**：侧栏没有新增空 Studio 会话

### T7：Studio 三步真实生图

1. 输入明确商品描述，生成三视图
2. **预期**：只创建一个 Studio 会话；正面、侧面、顶面视角明确不同
3. 输入外观调整要求
4. **预期**：只修改指定属性，其余商品外观和视图排版保持一致
5. 选择一个场景模板并生成场景图
6. **预期**：请求实际使用模板 Prompt；场景图中只出现一个商品
7. 刷新页面并切换其他会话后再返回
8. **预期**：参考图、当前阶段、调整历史和场景结果均保留

### T7.5：Studio 会话隔离与首次进入（2026-06-10 修复回归）

1. 先打开一个小红书聊天会话，再从欢迎页进入 Studio
2. **预期**：Studio 为全新空状态；生成后产生独立 studio 会话，原聊天会话不被写入 studio 状态或消息
3. 新对话 → 欢迎页 → 直接进入 Studio（无活跃会话），在描述框输入文字
4. **预期**：可以正常输入；点击生成后描述随新会话保留
5. 退出 Studio 再重新进入同一 Studio 会话，继续生成
6. **预期**：控制台无 React key 重复警告（消息 ID 使用 studio- 前缀）
7. 微调一张场景图的同时生成新场景图
8. **预期**：微调完成后新生成的场景图不丢失
9. 恢复一天前的 Studio 会话并执行调整/场景生成
10. **预期**：后端自动将过期 OSS URL 下载转 base64，生成成功

### T8：Studio 输入与失败边界

1. 不输入描述且不上传图片，尝试生成
2. **预期**：前后端均拒绝请求，不提交付费生图任务
3. 上传接近 5MB 的图片
4. **预期**：前后端使用一致的实际请求大小限制；超限时显示明确错误
5. 模拟上游任务失败或超时
6. **预期**：上游失败或后端非 2xx 时前端立即停止；总轮询时间有明确上限并显示失败信息

### T9：平台强契约与错误成品拦截（2026-06-10 单测覆盖：tests/test_platform_contract.py）

1. 在小红书会话模拟模型返回 Amazon 英文 Listing
2. **预期**：系统执行一次自动矫正；矫正前的错误成品不展示、不保存
3. 模拟自动矫正后仍返回错误平台结构
4. **预期**：返回明确失败提示或待补充草稿；不创建内容资产
5. 分别验证抖音、小红书、Amazon 的结构缺失和跨平台污染
6. **预期**：结果卡状态与后端真实校验结果一致，不错误显示“平台格式已适配”

### T10：流式与非流式能力一致（2026-06-10 真实 API 实测：sources 4 条、quality 80；禁忌词共用 post_process 由单测覆盖）

1. 在品牌档案中添加自定义禁忌词，分别调用流式和非流式生成
2. **预期**：两条链路均返回品牌禁忌词警告
3. 发起需要公开网页发现的生成请求
4. **预期**：流式和非流式响应均返回参考来源
5. 对同一模型结果比较两条链路
6. **预期**：平台校验、后处理、来源、资产保存结果一致

### T11：客服话术与平台枚举

1. 从欢迎页进入客服话术，生成结构化客服成品
2. **预期**：结果卡完整展示标题、正文和场景话术，不出现空白主体
3. 尝试让 `studio` 平台进入聊天生成接口
4. **预期**：系统在边界明确拒绝，或使用完整 Studio Prompt；不得出现 500/`KeyError`（2026-06-10 实测：两个 chat 端点均 422 ✓）

### T12：前端可访问性与移动端

1. 在 390×844 视口进入 Studio
2. **预期**：始终可返回欢迎页或打开侧栏导航（2026-06-10 浏览器实测：菜单→新对话→工作室→返回欢迎页全程可达 ✓）
3. 使用触屏方式删除会话
4. **预期**：无需 hover 即可找到删除入口，并弹出确认框
5. 仅使用键盘切换会话、聚焦删除、取消和确认
6. **预期**：焦点可见，所有操作可完成，删除控件不嵌套在另一个按钮内

### T13：构建与部署路由

1. 运行 `npm run lint`
2. **预期**：零 error
3. 运行 `npm run build`
4. **预期**：通过，且无静态导出与 rewrites 冲突警告
5. 分别在本地开发和生产部署路径调用 `/shopgenie/api/profile`、`/shopgenie/api/chat/stream`、`/shopgenie/api/studio/templates`
6. **预期**：均正确到达 FastAPI，不返回 Next.js 404（2026-06-10 本地开发 curl 实测均 200 ✓；生产静态导出仍不支持 rewrites，需部署层代理，待验）
