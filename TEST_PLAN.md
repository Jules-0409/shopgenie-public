# 交互逻辑测试用例

> 改动：选平台不立即建会话，发第一条消息后才出现在侧栏。

## 测试步骤

### T1：选平台不建会话

1. 打开 https://www.liujufu.com/shopgenie/
2. 点击「小红书种草笔记」卡片
3. **预期**：切到聊天界面，顶部显示「小红书」pill + 「新对话」
4. **预期**：左侧侧栏**没有**新增任何会话（只有演示会话）
5. **预期**：输入框可见，预填了 starter text

### T2：发消息后出现在侧栏

1. 接 T1，输入框已有文字，直接按 Enter 发送
2. **预期**：消息发出，开始生成
3. **预期**：左侧侧栏**立刻出现**新会话
4. **预期**：新会话标题从 LLM 返回的 conversation_title 更新

### T3：中途返回再选平台

1. 已有至少一个真实会话
2. 点「新对话」→ 回到欢迎页
3. 选「抖音短视频脚本」
4. **预期**：切到聊天界面，侧栏不变
5. 不发送，直接按「新对话」→ 回到欢迎页
6. **预期**：侧栏依然只有之前的会话，没有新的

### T4：切换已有会话

1. 欢迎页选平台但不发消息
2. 点侧栏中已有的某个会话
3. **预期**：正常切换到该会话，pending 状态清空
4. 再点「新对话」→ 回欢迎页
5. 选同样或另一个平台但不发消息
6. **预期**：正常切到空白聊天界面

### T5：连续发多条

1. 选平台 → 发第一条消息 → 会话出现在侧栏
2. 在同一个会话继续发第二条
3. **预期**：侧栏不变（不会重复创建）
4. 第三条、第四条同理

### T6：设计助手 + AI 生图按钮

1. 选「设计助手」
2. **预期**：顶部栏出现「AI 生图」按钮
3. 点 AI 生图 → 右侧滑出面板
4. 不发送消息，关闭面板，返回欢迎页
5. **预期**：侧栏没有新增会话
